### 3. SENSITIVE USER DATA HANDLING

#### **Data Classification**
```
🔴 HIGHLY SENSITIVE (Encrypt in Transit & At Rest):
   - Stripe Payment IDs & Tokens
   - Hashed IP Addresses
   - Email addresses (PII)
   - Full names (PII)
   - Membership history
   - Purchase history
   - Admin credentials

🟡 MODERATELY SENSITIVE (Encrypt at Rest):
   - Theme preferences
   - Notification settings
   - Referral codes
   - Social links
   - Avatar URLs

🟢 NOT SENSITIVE (Can be public):
   - Display names (pseudonyms)
   - Membership tier (current only)
   - Public profile information
   - Feature access list
```

#### **Firestore Security Rules**
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Users collection - read own data only
    match /users/{userId} {
      allow read: if request.auth.uid == userId;
      allow write: if request.auth.uid == userId && 
                     !request.resource.data.keys().hasAny(['isAdmin']);
      allow read, write: if isAdmin(request.auth.uid); // Admin override
    }
    
    // Purchases - read own, admin can read all
    match /purchases/{purchaseId} {
      allow read: if request.auth.uid == resource.data.userId ||
                     isAdmin(request.auth.uid);
      allow write: if request.auth.uid == resource.data.userId ||
                      isAdmin(request.auth.uid);
    }
    
    // Cart items - read/write own
    match /cartItems/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
    
    // Admin actions - admin only
    match /adminActions/{actionId} {
      allow read, write: if isAdmin(request.auth.uid);
    }
    
    // Feature access - read own
    match /featureAccess/{userId} {
      allow read: if request.auth.uid == userId ||
                     isAdmin(request.auth.uid);
      allow write: if isAdmin(request.auth.uid);
    }
    
    // Coupons - everyone can read active coupons
    match /coupons/{couponCode} {
      allow read: if resource.data.active == true;
      allow write: if isAdmin(request.auth.uid);
    }
    
    // Admin check function
    function isAdmin(uid) {
      return get(/databases/$(database)/documents/users/$(uid)).data.isAdmin == true;
    }
  }
}
```

#### **Data Encryption Strategy**
```
In Transit:
  ✅ HTTPS/TLS for all communication
  ✅ Firebase enforces HTTPS
  ✅ Sensitive fields: use field-level encryption

At Rest (Firestore):
  ✅ Firebase automatic encryption
  ✅ Field-level encryption for Stripe IDs:
     - Use: encryptStripeId(stripePaymentId)
     - Store as: stripePaymentId_encrypted
     - Decrypt only when needed for refunds

Client-Side (localStorage):
  ⚠️ NEVER store sensitive data in localStorage
  ✅ Only store: userId, theme, membershiptier (current)
  ✅ NOT stored: passwords, payment tokens, personal info
```
